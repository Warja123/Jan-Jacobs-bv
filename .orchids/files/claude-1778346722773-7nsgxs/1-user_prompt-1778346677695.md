Cause
- TypeScript does not know how to type-check plain CSS imports by default and there is no ambient module declaration (.d.ts) for '*.css' in the project, so tsc fails during the build.

Solution
1. Verify the CSS file exists and was committed
   - Confirm src/index.css is present in the repo and committed. If it is missing, add and commit it. (You can check in the repo UI: https://github.com/Warja123/Jan-Jacobs-bv/tree/master/src)

2. Add a TypeScript declaration for CSS imports
   - Create a file named src/env.d.ts (or src/global.d.ts) and add the following content:

```typescript
// src/env.d.ts
declare module '*.css';
declare module '*.scss';
declare module '*.sass';
declare module '*.less';
```

   - This tells TypeScript that importing .css (and common style extensions) is allowed.

3. Ensure the declaration file is included by tsc
   - If your tsconfig.json limits included files, make sure it includes the src directory (or the new file specifically). Example tsconfig snippet:

```json
{
  "compilerOptions": {
    // your existing compilerOptions
  },
  "include": ["src"]
}
```

   - Committing src/env.d.ts to the repo is usually enough because "include": ["src"] will pick it up.

4. Commit and redeploy
   - Commit the new declaration file and push to GitHub. Trigger a new Netlify deploy. The tsc error for the CSS import should be resolved.

Optional alternatives (choose one)
- Instead of adding env.d.ts, you can add "types": ["vite/client"] under "compilerOptions" in tsconfig.json if you rely on Vite and its client types; however, the simpler and more universal fix is the declaration file above.
